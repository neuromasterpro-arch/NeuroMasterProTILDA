## Как применяются изменения
Все изменения, которые делает агент, сразу записываются в этот файл и фиксируются отдельным коммитом. Их можно увидеть после `git pull` или в истории репозитория.
`git pull` обновляет вашу локальную копию репозитория: команда скачивает последние коммиты с удалённого сервера и сливает их в вашу текущую ветку. Если выполнить её на своём компьютере, в файлах появятся все изменения, которые агент уже зафиксировал здесь.

## Как внести и зафиксировать свои изменения
1. Отредактируйте нужные файлы в рабочей копии.
2. Добавьте их в индекс командой `git add <имя_файла>`.
3. Зафиксируйте правки: `git commit -m "Ваше описание"`.
4. Отправьте результат на сервер через `git push`, после чего другие пользователи смогут получить ваши изменения через `git pull`.

## Где вводить команды
Все команды (`git add`, `git commit`, `git push` и т.д.) выполняются в терминале:

- Откройте терминал на машине, где у вас находится локальная копия репозитория.
- Перейдите в корень проекта: `cd /путь/к/NeuroMasterProTILDA`.
- Вводите команды из предыдущего раздела в этом каталоге.

Если вы работаете через IDE с интегрированным терминалом, достаточно открыть терминал внутри IDE, убедиться, что текущая папка — корень репозитория, и запускать команды там.
